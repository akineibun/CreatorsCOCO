# Plan.md — CG Creator Studio（仮称）

創作向け文字入れ・モザイクツール 仕様書

---

## 1. プロジェクト概要

CG集・支援サイト（DLsite / Booth / Pixiv / FANZA / Patreon / Discord等）向けの作品制作を効率化するデスクトップアプリケーション。画像への文字入れ、モザイク処理、メッセージウインドウ配置、吹き出し生成を統合的に行える。

### ターゲットユーザー

- CG集制作者（DLsite / Booth / FANZA向け）
- 支援サイト運営者（Patreon / Fantia / Discord等）
- 同人・インディーゲーム制作者

### 解決する課題

- 複数ツールを行き来する非効率なワークフロー
- モザイク処理の手間（特に大量ページ）
- テンプレート的な作業の繰り返し（セリフ差し替え、ウォーターマーク追加等）

## 2. 技術スタック

### フロントエンド

| 項目 | 技術 |
|------|------|
| フレームワーク | Electron |
| UIフレームワーク | React + Vite |
| UIコンポーネント | shadcn/ui |
| キャンバス描画 | react-konva（Canvas 2D） + カスタムノード |
| 状態管理 | Zustand（推奨） |
| DB（プロジェクト管理） | SQLite（better-sqlite3 or electron-store） |

#### react-konvaの制約と対策

react-konvaは基本的な描画に適しているが、以下の機能はカスタム実装が必要。

| 機能 | 対策 |
|------|------|
| 縦書きテキスト | Canvas 2Dで1文字ずつ描画 → Konva.Image化するカスタムノード |
| グラデーション文字 | Canvas 2DのcreateLinearGradient + fillText |
| 2重フチ取り | Canvas 2DのstrokeTextを複数回描画（太い→細い順） |
| ルビ | 親文字の上部/右部に小サイズで描画するカスタムノード |
| 4K画像表示 | ビューポート内のみ描画 + 縮小表示時はリサイズ済み画像を使用 |

パフォーマンス問題が深刻な場合、将来的にPixiJS（WebGL）への移行を検討。

### バックエンド（Python）

| 項目 | 技術 |
|------|------|
| サーバ | FastAPI（ローカル常駐） |
| セグメンテーション | SAM3（Segment Anything Model 3） |
| NSFW検出 | NudeNet（YOLOベース） |
| 画像処理 | Pillow / OpenCV |
| パッケージング | PyInstaller（アプリ同梱用独自Python環境） |

#### Python-Electron間通信

- Electron起動時にFastAPIサーバをchild_processで起動
- REST APIで通信（localhost:ポート番号）
- SAM3モデルはサーバ起動時に1度だけロード（メモリ常駐）
- NudeNetモデルも同様にロード

#### モデル管理

- 初回起動時にSAM3 / NudeNetモデルを自動ダウンロード
- ダウンロード進捗をElectron側UIで表示
- モデルサイズ選択：SAM3-base（デフォルト）/ SAM3-large（高精度）
- GPU前提、CPUフォールバックは低優先度で実装

## 4. 機能仕様

### 4.1 プロジェクト管理

#### プロジェクト構造

- 1プロジェクト = 複数ページ（画像）
- プロジェクトファイルはアプリ内DB（SQLite）に保存
- 最近開いたプロジェクトの履歴表示
- ページ数上限なし（200ページ以上を想定）

#### 自動保存

- 300秒（5分）間隔で自動保存
- 保存中のインジケーター表示

#### アンドゥ・リドゥ

- 操作履歴：50ステップ（目安、メモリ使用量に応じて調整）
- Ctrl+Z / Ctrl+Y対応

#### ショートカットキー

- 基本操作のみ：保存（Ctrl+S）、アンドゥ（Ctrl+Z）、リドゥ（Ctrl+Y）、ツール切替（キーボード1文字割当）
- カスタマイズは将来対応

### 4.2 画像入出力

#### 入力

- 対応フォーマット：PNG / JPEG / WebP
- クリップボードからの貼り付け対応
- 最大解像度：4K（3840×2160）
- ドラッグ&ドロップで画像追加

#### 出力

- PNG透過出力（メイン出力）
- 画像圧縮品質選択（販売プラットフォーム対応）
- 書き出し時にEXIF/メタデータ全削除
- 画像サイズ統一機能（余白追加・リサイズで全ページ同一サイズに）

#### ZIP出力

- 連番ファイル名（001.png, 002.png ...）
- カスタムテンプレート（`{prefix}_{number}` 等）
- ファイル名規則の切替UI

#### PDF出力

- 画像サイズそのままで1ページ1画像
- 画像圧縮品質選択

### 4.3 文字入れ機能

#### テキスト基本設定

- フォント選択（システムフォント + ユーザー追加フォルダ監視）
- フォントサイズ（px指定、スライダー + 数値入力）
- 行間（line-height、スライダー + 数値入力）
- 文字間隔（letter-spacing）
- テキストボックス最大幅指定 → 自動改行
- 文字色（単色 / グラデーション）
- 透明度

#### 横書き・縦書き

- 横書き：標準テキスト描画
- 縦書き：
  - 1文字ずつ縦に配置するカスタム描画
  - 英数字は全角に変換
  - 「。」「、」「）」等の約物位置補正
  - 縦中横は将来対応
  - 最低限の禁則処理（行頭禁止文字）

#### ルビ（ふりがな）

- 親文字の上部（横書き）/ 右部（縦書き）に配置
- ルビフォントサイズ：親文字の50%（デフォルト）
- ルビ位置オフセット微調整（上下/左右）
- ルビにも親テキストと同一のエフェクト一括適用

#### テキストエフェクト（テキスト単位で適用）

| エフェクト | 詳細 |
|-----------|------|
| フチ取り | 1重 / 2重対応、各フチの色・太さ指定 |
| ぼかしフチ | フチ取りにガウスぼかし適用 |
| ドロップシャドウ | 方向・距離・ぼかし・色 |
| グラデーション文字色 | 線形グラデーション、角度指定 |
| グラデーションフチ | フチにもグラデーション適用 |
| 背景帯 | テキスト背後に半透明の矩形帯、色・透明度・パディング指定 |
| 回転 | 自由角度での斜め配置（可能なら実装） |

#### フォントプレビュー

- システムフォント自動取得 + ユーザー追加フォントフォルダの監視
- フォント一覧でプレビュー表示（サンプルテキストをそのフォントで表示）
- お気に入りフォント機能（ピン留め → 一覧上部に表示）
- カテゴリ自動分類（ゴシック / 明朝 / 手書き / デコ等、フォントメタデータベース）

### 4.4 メッセージウインドウ

#### 構成

- メインテキスト領域：本文表示用、フォント設定独立
- サブテキスト領域：キャラ名・所属表示用、位置指定可能、フォント設定独立
- 各テキスト領域に4.3のテキストエフェクト全て適用可能

#### ウインドウ素材

- 9-slice方式で引き伸ばし（四隅固定、辺・中央を伸縮）
- オリジナルPNG素材の読み込み対応
- ウインドウ透明度：独立設定（0〜100%）

#### 配置

- デフォルト：画面下部
- 自由配置モード：ドラッグで任意の位置に
- サイズ調整：幅・高さを個別に指定

#### プリセット

- ウインドウ設定（素材 + テキスト設定 + 位置 + サイズ + 透明度）をプリセット保存
- プリセットの呼び出し
- 他のページから同じ設定をコピー

### 4.5 黒背景オーバーレイ

- 画像の上に半透明のカラーオーバーレイを配置
- オーバーレイ色：任意の色を選択可能（デフォルト黒）
- 透明度：スライダーで調整
- 範囲：画面全体 / 上半分 / 下半分 / 帯状（Y座標指定）
- グラデーションオーバーレイ：方向（上→下、下→上等）と透明度のグラデーション
- オーバーレイの上にテキスト配置（4.3のテキスト機能をそのまま利用）

### 4.6 モザイク処理

#### モザイクの種類

| 種類 | 説明 |
|------|------|
| ピクセルモザイク | ブロック単位でピクセル化 |
| ガウスぼかし | ガウシアンブラー |
| 白塗り | 選択領域を白で塗りつぶし |
| 黒塗り | 選択領域を黒で塗りつぶし |
| カスタム画像スタンプ | PNG素材で塗りつぶし（ハートやスタンプ等） |

#### モザイク強度

- 小 / 中 / 大 の3段階プリセット
- Pixiv基準プリセット（Pixivのガイドラインに準拠した強度）
- 各プリセットのブロックサイズは設定画面で調整可能

#### 選択方法

- 矩形選択
- フリーハンド（投げ縄）選択
- SAM3によるクリック自動選択（4.7参照）

#### 操作

- リアルタイムプレビュー（適用前に結果を確認）
- 適用後も再調整可能（非破壊編集）

### 4.7 SAM3 + 自動NSFW検出

#### 自動検出フロー

1. NudeNet（YOLOベース）で規制対象領域を自動検出
   - 検出対象：女性器、男性器、肛門（内臓部分が見えている場合）
2. 検出結果のバウンディングボックスをUI上に表示
3. ユーザーが確認・承認
4. SAM3で各検出領域の正確なセグメンテーションマスクを生成
5. マスク領域にモザイク処理を適用

#### 手動SAM3モード

- 画像上をクリック → SAM3がクリック地点を含むオブジェクトを自動セグメント
- ポジティブクリック（含める）/ ネガティブクリック（除外する）で精度調整
- 生成されたマスクにモザイク処理を適用

#### 手動補正

- マスクレイヤーを生成（元画像のコピーにモザイク適用した状態）
- マスクに対してブラシ追加（モザイク範囲を広げる）
- 消しゴムツール（モザイク範囲を削る）
- 補正後のマスクでモザイク再適用

#### 注意事項

- 自動検出の精度は100%ではない。必ずユーザーによる確認ステップを入れる
- 検出結果が不適切な場合（誤検出・未検出）は手動で追加/削除
- GPU使用前提。CPUフォールバックは低優先度

### 4.8 吹き出し

#### 形状バリエーション

| 形状 | 説明 | ランダマイズ |
|------|------|-------------|
| 丸 | 標準的な丸い吹き出し | ○（楕円の歪み） |
| 角丸四角 | 角丸長方形 | - |
| トゲ（叫び） | 星形に近い鋭角の突起 | ○（突起の数・角度・長さ） |
| もくもく（思考） | 雲のような丸い突起 | ○（突起の数・サイズ） |
| ウニ（強い叫び） | さらにトゲトゲした形状 | ○（突起パラメータ） |
| 破線（ヒソヒソ声） | 破線で描かれた吹き出し | - |
| 四角（解説・ナレーション） | 直線的な四角形 | - |

- トゲ / もくもく / ウニは生成のたびにランダムパラメータで形状が変わる
- リロールボタンでランダム再生成
- カスタムPNG素材の読み込み対応

#### 尻尾

- 吹き出し生成時に自動付与
- 尻尾部分をドラッグで方向・位置を手動調整
- 尻尾の長さ・太さも調整可能

#### テキスト

- 吹き出し内テキストに4.3のテキストエフェクト全て適用可能
- テキスト量に応じて吹き出しサイズを自動調整
- 手動でサイズ固定も可能

#### 自動配置（SAM3流用）

- SAM3で画像内のオブジェクト領域を検出
- オブジェクトが少ない空きスペースを算出
- 空きスペースに吹き出しを自動配置
- 配置後はドラッグで位置調整可能

#### その他

- 吹き出し透明度設定
- 吹き出しの線の太さ・色設定
- 吹き出し背景色設定

### 4.9 レイヤー管理

#### レイヤーの種類

- 画像レイヤー（ベース画像）
- テキストレイヤー
- メッセージウインドウレイヤー
- 吹き出しレイヤー
- モザイクレイヤー
- オーバーレイレイヤー（黒背景等）
- ウォーターマークレイヤー
- 画像素材レイヤー（配置したPNG素材等）

#### 操作

- 表示 / 非表示の切替（目のアイコン）
- ドラッグでレイヤー順序の並べ替え
- レイヤーのロック（誤操作防止）
- レイヤーのグループ化（吹き出し+テキストを1グループ等）
- レイヤー選択 → キャンバス上の対応要素がハイライト

### 4.10 テンプレート機能

#### テンプレートに含まれる設定

- テキスト設定（フォント・サイズ・色・エフェクト・位置）
- メッセージウインドウ設定（素材・テキスト設定・位置・サイズ）
- オーバーレイ設定（色・透明度・範囲）
- モザイク設定（種類・強度・位置）
- ウォーターマーク設定
- 配置した画像素材

#### 操作

- 現在のページ設定をテンプレートとして保存
- テンプレートを他のページに一括適用
- ページ複製 → テキストのみ差し替え（CG集のセリフ差し替え用）
- テンプレートのサムネイルプレビュー
- 一度作った構成をPNG画像化して素材として保存・再利用

### 4.11 プリセット管理（横断）

- 全てのプリセット（テキスト設定、ウインドウ設定、テンプレート等）を一元管理
- プリセット名の編集
- プリセットの削除
- プリセットの複製
- 作業中のページから設定をコピーして他ページに適用

## 4.12 UI/UX全面改善と実用レベルへの再設計

### 背景

- 現状のUIは機能接続は進んでいるが、実利用時の視認性・操作性・学習コストの面で未完成。
- 画像の上に説明テキストや補助表示が重なり、編集対象そのものを見失いやすい。
- テキスト、吹き出し、会話枠、オーバーレイなどの見た目が仮表示寄りで、実際の仕上がりを判断しづらい。
- サイズ変更や位置調整がボタン中心で、一般的な編集ソフトの操作期待に合っていない。
- 全体として「動作確認はできるが、常用できるUI」には至っていない。

### 目標

- UIを全面的に見直し、少なくとも身内テストで「迷わず使える」水準まで引き上げる。
- 中央キャンバスを主役にし、画像確認と編集結果確認を最優先にしたレイアウトへ再設計する。
- 他の画像編集・漫画編集・配信用クリエイティブツールを参考に、馴染みやすいUI/UXを採用する。
- 仮表示ではなく、編集結果に近いプレビューを標準にする。

### 再設計方針

- キャンバス上に常時重なる説明テキスト・補助カードは最小化し、必要時のみ表示する。
- 左右パネル、上部バー、下部ステータスの責務を整理し、「何をする場所か」が一目で分かる構成にする。
- ツールはアイコン中心でもよいが、hover / 選択状態 / 現在モードが明確に分かることを必須条件とする。
- 主要操作は日本語で統一しつつ、専門用語は必要に応じて補足する。
- 頻出操作はボタン連打ではなく、入力欄・数値ボックス・スライダー・ドラッグ操作を中心に再設計する。

### 具体タスク

- 主要商用/定番ツールのUI調査
  - 参考対象: Photoshop, CLIP STUDIO PAINT, Canva, Figma, comic/template editor系ツール
  - レイアウト、ツール配置、インスペクター、プロパティ編集、レイヤー一覧、ズーム導線を比較
- 情報設計のやり直し
  - グローバル操作、ページ操作、レイヤー操作、プロパティ編集、書き出し設定を分離
  - 「今選択中の対象」と「今変更できる値」を常に分かるようにする
- キャンバスUIの再設計
  - 画像の上に不要なオーバーレイを出さない
  - テキスト・吹き出し・会話枠・透かし・モザイクを実表示寄りに改善
  - 選択枠、ハンドル、スナップ、整列表示を見やすく整理
- プロパティ編集UIの刷新
  - 位置、サイズ、透明度、角度、フォントサイズなどを数値入力とスライダーで編集可能にする
  - ボタンでしか変更できない項目を減らす
  - テキストボックス / スピンボックス / セグメント切替 / カラーピッカーの使い分けを整理
- モード別UXの整理
  - 選択 / 文字 / 会話枠 / 吹き出し / モザイク / オーバーレイで表示すべきプロパティを最適化
  - 未選択時は簡潔、選択時は詳細を出す
- 初回利用導線の改善
  - 画像を読み込んだ後の最初の一手が直感的に分かるUIにする
  - サンプル画像読み込み、保存、書き出し、レイヤー追加までの導線を短くする

### 完了条件

- キャンバスを見たときに、画像が主役でありUIに隠されない
- テキスト・吹き出し・会話枠が「仮ラベル」ではなく、編集結果に近い見た目で確認できる
- サイズや位置調整の主要操作が入力欄またはスライダー中心で行える
- 初見ユーザーがサンプル画像読み込み→文字追加→保存→PNG書き出しまで迷わず到達できる
- 身内テストで「見た目が荒い」「どこを触ればいいか分からない」という指摘が大幅に減る
